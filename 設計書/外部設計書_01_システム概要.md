# 外部設計書 - システム概要

## 1. システム概要

### 1.1 システム名
Compact Bayesian Model GUI

### 1.2 目的
PyMCを用いたベイズ統計モデルをGUIで視覚的に構築し、推論を実行できるWebアプリケーションを提供する。

### 1.3 システムの特徴
- **ノードベースのモデル構築**: ドラッグ&ドロップで確率変数と依存関係を定義
- **リアルタイムバリデーション**: モデル構築時に制約違反を即座に検出
- **自動コード生成**: 構築したモデルからPyMCコードを自動生成
- **統計推論の実行**: MCMC・変分推論による事後分布の推定
- **可視化機能**: トレースプロット、事後分布、診断指標の表示

### 1.4 想定ユーザー
- ベイズ統計に興味がある研究者・データサイエンティスト
- PyMCの学習者
- プログラミングに不慣れな統計解析者

### 1.5 主要機能
1. **モデル構築機能**
   - グラフィカルなノード配置とエッジ接続
   - 確率分布の選択とパラメータ設定
   - 決定論的ノードによる計算式定義
   - CSVデータのインポートと観測データの紐付け

2. **バリデーション機能**
   - 循環依存の検出
   - パラメータ制約の検証
   - 型の整合性チェック

3. **コード生成機能**
   - PyMCコードの自動生成
   - トポロジカルソートによる依存順の解決

4. **推論実行機能**
   - MCMC（NUTS/Metropolis）サンプリング
   - 変分推論（ADVI/FullRank ADVI）
   - 事前予測チェック

5. **結果可視化機能**
   - トレースプロット
   - 事後分布の可視化
   - 診断指標（R-hat、ESS）の表示

### 2.2 技術スタック

#### フロントエンド
- **フレームワーク**: React 18 + TypeScript
- **ルーティング**: React Router v6
- **状態管理**: Zustand
- **UIライブラリ**: Chakra UI
- **グラフ描画**: React Flow
- **データフェッチング**: TanStack Query (React Query)
- **ビルドツール**: Vite

#### バックエンド
- **フレームワーク**: FastAPI
- **統計エンジン**: PyMC 5.x
- **数値計算**: NumPy, Pandas
- **可視化**: ArviZ, Matplotlib
- **バリデーション**: Pydantic

## 3. データフロー

### 3.1 モデル構築フロー

```
1. ユーザー操作
   ↓
2. フロントエンド状態更新 (Zustand)
   ↓
3. リアルタイムバリデーション
   ↓
4. バックエンドAPI呼び出し（保存時）
   ↓
5. サーバー側バリデーション
   ↓
6. モデルデータ保存
```

### 3.2 推論実行フロー

```
1. 推論設定入力
   ↓
2. モデルデータ + 推論設定をAPI送信
   ↓
3. バックエンドでバリデーション
   ↓
4. PyMCモデル構築
   ↓
5. サンプリング/変分推論実行
   ↓
6. 結果の可視化処理（ArviZ）
   ↓
7. 結果データ + プロット画像を返却
   ↓
8. フロントエンドで結果表示
```

### 3.3 データインポートフロー

```
1. CSVファイル選択
   ↓
2. ファイルアップロード（マルチパート）
   ↓
3. サーバーで一時保存
   ↓
4. 列情報の解析
   ↓
5. メタデータ返却
   ↓
6. データノード生成
   ↓
7. 観測データとして紐付け
```

## 4. 非機能要件

### 4.1 パフォーマンス
- モデル構築操作のレスポンス: 100ms以内
- 推論実行時間: モデル複雑度とサンプル数に依存（通常数秒〜数分）
- 同時接続ユーザー: 最大10名（研究室内利用を想定）

### 4.2 セキュリティ
- ファイルアップロード: CSV形式のみ許可、サイズ制限10MB
- 式評価: 制限された名前空間でのeval実行
- CORS: 開発環境のみ有効

### 4.3 可用性
- 単一インスタンス構成（研究用途）
- セッション管理なし（ステートレス）

### 4.4 保守性
- コードの可読性: 1ファイル300行以下を目標
- テストカバレッジ: 主要ロジックのユニットテスト
- ドキュメント: 主要機能にコメント記載

## 5. 制約事項

### 5.1 技術的制約
- PyMCバージョン: 5.x系に依存
- ブラウザ: モダンブラウザ（Chrome, Firefox, Safari）のみサポート
- データサイズ: メモリに収まる範囲（推奨100MB以下）

### 5.2 機能的制約
- モデルの永続化: ローカルストレージのみ（サーバー側保存なし）
- 推論の並列実行: 不可（1リクエストずつ処理）
- リアルタイム進捗表示: サポートなし（完了まで待機）
