# 内部設計書 - データモデル

## 1. データモデル全体図

```
PyMCModel
├── nodes: List[Node]
│   ├── VariableNode (type: "variable")
│   ├── ComputedNode (type: "computed")
│   └── DataNode (type: "data")
├── edges: List[Edge]
└── csvMetadata: Optional[CsvMetadata]
```

## 2. コアモデル

### 2.1 PyMCModel

**説明**: ベイズモデル全体を表現するルートモデル

**TypeScript定義**
```typescript
interface PyMCModel {
  nodes: Node[]
  edges: Edge[]
  csvMetadata?: CsvMetadata
}
```

**Python定義（Pydantic）**
```python
class PyMCModel(BaseModel):
    nodes: List[Node]
    edges: List[Edge]
    csv_metadata: Optional[Dict[str, Any]] = None
```

**フィールド説明**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| nodes | List[Node] | ✓ | モデルを構成するノードのリスト |
| edges | List[Edge] | ✓ | ノード間の依存関係を表すエッジのリスト |
| csvMetadata | CsvMetadata? | - | インポートされたCSVデータのメタ情報 |

**検証ルール**
- nodesは空でも可（新規モデル時）
- edgesは空でも可
- 同一IDのノードは許可しない
- 存在しないノードを参照するエッジは許可しない

### 2.2 Node

**説明**: モデルの構成要素（確率変数、決定論的変数、データ）

**TypeScript定義**
```typescript
enum NodeType {
  VARIABLE = 'variable',
  COMPUTED = 'computed',
  DATA = 'data'
}

interface Node {
  id: string
  type: NodeType
  name: string
  position: { x: number; y: number }
  distribution?: string
  parameters: Record<string, any>
  shape: number[] | null
  observed: boolean
  observedDataSource?: string
  expression?: string
}
```

**Python定義**
```python
class Node(BaseModel):
    id: str
    type: str  # "variable", "computed", "data"
    name: str
    position: Dict[str, float]
    distribution: Optional[str] = None
    parameters: Dict[str, Any] = {}
    shape: Optional[List[int]] = None
    observed: bool = False
    observed_data_source: Optional[str] = None
    expression: Optional[str] = None
```

**フィールド説明**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| id | string | ✓ | ノードの一意識別子（UUID形式） |
| type | NodeType | ✓ | ノードの種別（variable/computed/data） |
| name | string | ✓ | 変数名（PyMCコード生成時に使用） |
| position | {x, y} | ✓ | キャンバス上の配置座標 |
| distribution | string? | - | 確率分布名（variableノードのみ） |
| parameters | Record<string, any> | ✓ | 分布のパラメータまたは計算式の変数 |
| shape | number[]? | - | 配列の形状（多次元確率変数用） |
| observed | boolean | ✓ | 観測データとして扱うか |
| observedDataSource | string? | - | 観測データのソース（DataノードID） |
| expression | string? | - | 決定論的計算式（computedノードのみ） |

**ノードタイプ別の必須フィールド**

**Variableノード（確率変数）**
- 必須: id, type, name, position, distribution, parameters
- オプション: shape, observed, observedDataSource

**Computedノード（決定論的）**
- 必須: id, type, name, position, expression
- parameters: 式で参照される変数を格納（現在未使用）

**Dataノード**
- 必須: id, type, name, position, parameters
- parameters.column: CSVの列名を格納

**検証ルール**
- nameは有効なPython変数名であること（英数字とアンダースコア）
- distributionはDISTRIBUTIONSに定義された分布名であること
- parametersは分布の必須パラメータを全て含むこと
- observedがtrueの場合、observedDataSourceが必須
- computedノードの場合、expressionが必須

### 2.3 Edge

**説明**: ノード間の依存関係を表現

**TypeScript定義**
```typescript
interface Edge {
  id: string
  source: string
  target: string
}
```

**Python定義**
```python
class Edge(BaseModel):
    id: str
    source: str
    target: str
```

**フィールド説明**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| id | string | ✓ | エッジの一意識別子 |
| source | string | ✓ | エッジの始点ノードID |
| target | string | ✓ | エッジの終点ノードID |

**意味**
- source → target: sourceがtargetに影響を与える（依存関係）
- 例: `mu → y` の場合、yはmuに依存する

**検証ルール**
- sourceとtargetは存在するノードIDであること
- 同一ノードへの自己参照は許可しない（source ≠ target）
- 循環依存を形成しないこと

## 3. データ関連モデル

### 3.1 CsvMetadata

**説明**: インポートされたCSVファイルのメタデータ

**TypeScript定義**
```typescript
interface CsvMetadata {
  file_id: string
  filename: string
  columns: ColumnInfo[]
}
```

**Python定義**
```python
# Pydanticスキーマは未定義（Dictとして扱う）
# 内部的には以下の構造
{
    "file_id": str,
    "filename": str,
    "columns": List[ColumnInfo]
}
```

**フィールド説明**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| file_id | string | ✓ | サーバー上のファイル識別子（UUID） |
| filename | string | ✓ | 元のファイル名 |
| columns | ColumnInfo[] | ✓ | 列情報のリスト |

### 3.2 ColumnInfo

**説明**: CSV列のメタデータ

**TypeScript定義**
```typescript
interface ColumnInfo {
  name: string
  dtype: string
  role: 'observed' | 'feature' | 'unused'
}
```

**Python定義**
```python
{
    "name": str,
    "dtype": str,
    "role": str  # "observed", "feature", "unused"
}
```

**フィールド説明**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| name | string | ✓ | 列名 |
| dtype | string | ✓ | データ型（Pandasのdtype） |
| role | string | ✓ | データの役割 |

**role の意味**
- `observed`: 観測データ（目的変数） → モデルのobservedパラメータに渡される
- `feature`: 説明変数（共変量） → pm.Dataとして登録
- `unused`: 未使用

## 4. 推論関連モデル

### 4.1 InferenceRequest

**説明**: 推論実行のリクエストデータ

**TypeScript定義**
```typescript
interface InferenceRequest {
  model: PyMCModel
  config: InferenceConfig
}
```

**Python定義**
```python
class InferenceRequest(BaseModel):
    model: PyMCModel
    config: InferenceConfig
```

### 4.2 InferenceConfig

**説明**: 推論実行のパラメータ設定

**TypeScript定義**
```typescript
interface InferenceConfig {
  method: 'MCMC' | 'VI'

  // MCMC用
  sampler?: 'NUTS' | 'Metropolis'
  draws?: number
  tune?: number
  chains?: number
  cores?: number

  // VI用
  vi_method?: 'ADVI' | 'FullRankADVI'
  iterations?: number

  // 共通
  hdi_prob: number
}
```

**Python定義**
```python
class InferenceConfig(BaseModel):
    method: str
    sampler: Optional[str] = "NUTS"
    draws: int = 1000
    tune: Optional[int] = 1000
    chains: Optional[int] = 4
    cores: Optional[int] = 4
    vi_method: Optional[str] = "ADVI"
    iterations: Optional[int] = 10000
    hdi_prob: float = 0.95
```

**フィールド説明（MCMC）**

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| method | string | - | 推論手法（"MCMC"） |
| sampler | string | "NUTS" | サンプラー（NUTS/Metropolis） |
| draws | number | 1000 | 各チェーンのサンプル数 |
| tune | number | 1000 | チューニングサンプル数 |
| chains | number | 4 | マルコフ連鎖の数 |
| cores | number | 4 | 並列実行コア数 |
| hdi_prob | number | 0.95 | HDI区間の確率 |

**フィールド説明（VI）**

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| method | string | - | 推論手法（"VI"） |
| vi_method | string | "ADVI" | VI手法（ADVI/FullRankADVI） |
| iterations | number | 10000 | 最適化の反復回数 |
| draws | number | 1000 | 近似分布からのサンプル数 |
| hdi_prob | number | 0.95 | HDI区間の確率 |

## 5. バリデーション結果モデル

### 5.1 ValidationResult

**説明**: モデルバリデーションの結果

**TypeScript定義**
```typescript
interface ValidationResult {
  valid: boolean
  errors: ValidationError[]
  warnings: ValidationWarning[]
}

interface ValidationError {
  node_id?: string
  type: string
  message: string
}

interface ValidationWarning {
  node_id?: string
  type: string
  message: string
}
```

**Python定義**
```python
{
    "valid": bool,
    "errors": List[Dict[str, Any]],
    "warnings": List[Dict[str, Any]]
}
```

**エラータイプ**
- `cycle`: 循環依存
- `invalid_distribution`: 未知の分布
- `missing_parameter`: 必須パラメータ欠如
- `type_mismatch`: 型の不一致
- `constraint_violation`: 制約違反
- `invalid_shape`: 無効な形状

## 6. 分布定義モデル

### 6.1 DistributionInfo

**説明**: 確率分布の定義情報

**TypeScript定義**
```typescript
interface DistributionInfo {
  description: string
  params: Record<string, ParameterMeta>
  support: 'continuous' | 'discrete'
}

interface ParameterMeta {
  type: 'int' | 'float' | 'array'
  description: string
  default: any
  required: boolean
  constraint?: string
  example?: string
}
```

**Python定義**
```python
DISTRIBUTIONS = {
    "Normal": {
        "description": "Normal distribution",
        "params": {
            "mu": {
                "type": "float",
                "description": "Mean",
                "default": 0,
                "required": True,
                "constraint": None,
                "example": "0"
            },
            "sigma": {
                "type": "float",
                "description": "Standard deviation",
                "default": 1,
                "required": True,
                "constraint": "> 0",
                "example": "1"
            }
        },
        "support": "continuous"
    }
}
```

**フィールド説明（ParameterMeta）**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| type | string | ✓ | パラメータの型（int/float/array） |
| description | string | ✓ | パラメータの説明 |
| default | any | ✓ | デフォルト値 |
| required | boolean | ✓ | 必須パラメータか |
| constraint | string? | - | 制約式（例: "> 0"） |
| example | string? | - | 入力例 |

## 7. パラメータ値の表現

### 7.1 値の種類

パラメータ値は以下の3種類で表現される：

**1. 数値（スカラー）**
```typescript
parameters: {
  mu: "0",        // 文字列として保持
  sigma: "1.5"
}
```

**2. 配列**
```typescript
parameters: {
  alpha: [1, 1, 1],           // 1次元配列
  cov: [[1, 0], [0, 1]]       // 2次元配列
}
```

**3. ノード参照**
```typescript
parameters: {
  mu: "@prior_mu",    // @で始まる文字列
  sigma: "@prior_sigma"
}
```

### 7.2 パラメータの変換フロー

```
フロントエンド（入力）
  ↓
文字列 or 配列 or 参照
  ↓
バックエンド（コード生成）
  ↓
PyMCコード文字列
  ↓
バックエンド（モデル構築）
  ↓
Python数値/配列/変数オブジェクト
```

**例**:
1. 入力: `"1.5"` → コード: `sigma=1.5` → 実行: `float(1.5)`
2. 入力: `[1, 2]` → コード: `alpha=[1, 2]` → 実行: `[1, 2]`
3. 入力: `"@mu"` → コード: `sigma=mu` → 実行: `node_vars['mu']`

## 8. データフロー図

### 8.1 モデル構築フロー

```
User Input
  ↓
[Frontend] Node/Edge追加・編集
  ↓
[Zustand] modelStore更新
  ↓
[LocalStorage] 永続化
  ↓
[API] POST /api/validate
  ↓
[Backend] ModelValidator
  ↓
[Response] ValidationResult
  ↓
[Frontend] エラー/警告表示
```

### 8.2 推論実行フロー

```
[Frontend] InferenceConfig入力
  ↓
[API] POST /api/inference/run
  ↓
[Backend] InferenceEngine
  ↓
[PyMC] モデル構築
  ↓
[PyMC] サンプリング/VI実行
  ↓
[ArviZ] 結果の可視化・統計量計算
  ↓
[Response] InferenceResult
  ↓
[Frontend] 結果表示
```

## 9. データ永続化

### 9.1 フロントエンド

**LocalStorage**
- キー: `bayesian-model-builder`
- 値: JSON.stringify(PyMCModel)
- 保存タイミング: ノード/エッジ追加・更新時

### 9.2 バックエンド

**一時ファイル**
- CSVファイル: `/tmp/bayesian_gui_uploads/{file_id}.csv`
- ライフサイクル: セッション終了後も保持（手動削除が必要）

**永続化なし**
- モデルデータ: サーバー側では保存しない
- 推論結果: リクエストごとに再計算

## 10. 制約と限界

### 10.1 データサイズ

- CSVファイル: 最大10MB
- ノード数: 理論上無制限（実用上100ノード程度）
- エッジ数: 理論上無制限

### 10.2 型制約

- ノードname: Python変数名規則に従う（英数字+アンダースコア）
- パラメータ値: 文字列、数値、配列のみ
- 配列: ネスト可能（多次元対応）

### 10.3 バリデーション制約

- 循環依存: 検出・禁止
- 型不一致: 検出・警告（一部エラー）
- 制約違反: 検出・エラー（サポートされた制約のみ）
